# Dynamic Sitemap System

This news site implements a comprehensive dynamic sitemap system that automatically updates when content changes, ensuring optimal SEO performance and search engine indexing.

## Features

### 🚀 Dynamic Generation
- **Real-time Updates**: Sitemap automatically regenerates when articles are published, updated, or deleted
- **Multiple Sitemaps**: Separate sitemaps for different content types (main, news, categories)
- **Sitemap Index**: Central index file that references all sitemaps
- **Smart Caching**: Optimized caching strategies for performance

### 📊 SEO Optimization
- **Priority Calculation**: Dynamic priority assignment based on article metrics
- **Change Frequency**: Intelligent frequency settings based on content type
- **Google News Support**: Dedicated news sitemap for Google News inclusion
- **Search Engine Notifications**: Automatic ping to Google and Bing when content updates

### 🔄 Automation
- **Webhook Integration**: Real-time updates via webhook system
- **Scheduled Regeneration**: Automatic hourly sitemap updates
- **Bulk Operations**: Efficient handling of multiple article operations
- **Error Recovery**: Graceful fallback when database operations fail

## Sitemap Structure

### Main Sitemap (`/sitemap.xml`)
Generated by `app/sitemap.ts` using Next.js 14 native sitemap functionality.

**Includes:**
- Static pages (home, about, contact, etc.)
- All published articles with dynamic priorities
- Category pages
- Tag pages (popular tags only)
- Author pages
- Monthly archive pages

**Priority Calculation:**
```typescript
Base Priority: 0.6
+ Breaking News: +0.2
+ Featured Article: +0.1
+ High Views (>10k): +0.1
+ Recent (≤1 day): +0.1
+ Recent (≤7 days): +0.05
```

### News Sitemap (`/sitemap-news.xml`)
Specialized sitemap for Google News compliance.

**Features:**
- Only includes articles from last 2 days
- Google News XML schema compliance
- Publication metadata
- Keywords and categories
- Higher priority for breaking news

### Categories Sitemap (`/sitemap-categories.xml`)
Dedicated sitemap for category pages.

**Includes:**
- All news categories
- Dynamic last modified dates
- Appropriate change frequencies
- Priority based on category importance

### Sitemap Index (`/sitemap-index.xml`)
Central index referencing all sitemaps.

## Implementation

### Core Components

#### 1. Dynamic Sitemap Service (`lib/dynamic-sitemap.ts`)
Main service handling sitemap operations:

```typescript
import { dynamicSitemap } from '@/lib/dynamic-sitemap'

// Trigger updates
await dynamicSitemap.onArticlePublished(article)
await dynamicSitemap.onArticleUpdated(article)
await dynamicSitemap.onArticleDeleted(slug, category)
await dynamicSitemap.regenerateSitemap()
```

#### 2. Database Integration
Automatic sitemap updates integrated into database operations:

```typescript
// Auto-triggers sitemap update when article is saved as published
await saveArticle({ ...article, status: 'published' })

// Auto-triggers sitemap update when article is published
await publishArticle(articleId)
```

#### 3. Webhook System (`lib/sitemap-webhook.ts`)
Real-time webhook processing for external integrations:

```typescript
// Handle webhook events
await sitemapWebhook.handleWebhook({
  event: 'article.published',
  data: { article },
  timestamp: new Date().toISOString(),
  source: 'ai-editor'
})
```

### API Endpoints

#### Manual Sitemap Updates
```bash

{
  "action": "regenerate"

# Update for published article

{

# Update for deleted article

{
  "action": "article_deleted",
  "articleSlug": "article-slug",
  "category": "technology"

```

#### Webhook Endpoint

```bash
{
  "event": "article.published",
  "data": { "article": { ... } },

```

### Search Engine Integration

#### Automatic Notifications
The system automatically notifies search engines when content changes:

- **Google**: Ping sitemap URL
- **Bing**: Ping sitemap URL  
- **Google Search Console**: Submit individual URLs (if configured)
- **Bing Webmaster**: Submit individual URLs (if configured)

#### Configuration
Set environment variables for enhanced search engine integration:

```env
# Required
NEXT_PUBLIC_SITE_URL=https://your-news-site.com

# Optional - for enhanced search engine integration
GOOGLE_SEARCH_CONSOLE_TOKEN=your_token
BING_WEBMASTER_API_KEY=your_key
WEBHOOK_SECRET=your_webhook_secret
SITEMAP_WEBHOOK_URLS=https://external-service.com/webhook
```

## Performance Considerations

### Caching Strategy
- **Main Sitemap**: 1 hour cache
- **News Sitemap**: 1 hour cache (frequent updates for breaking news)
- **Categories Sitemap**: 2 hours cache (less frequent changes)
- **Sitemap Index**: 1 hour cache

### Rate Limiting
- Search engine pings are throttled to prevent spam
- Bulk operations are processed in batches
- Webhook processing includes retry logic

### Error Handling
- Graceful fallback to static pages if database fails
- Continued operation even if search engine notifications fail
- Comprehensive logging for debugging

## Best Practices

### For News Sites
1. **Frequent Updates**: News sitemaps should update every hour or when breaking news is published
2. **Priority Management**: Breaking news gets highest priority (1.0)
3. **Google News Compliance**: Use dedicated news sitemap with proper schema
4. **Archive Management**: Include monthly archives for historical content

### For SEO
1. **Consistent URLs**: Ensure article URLs remain stable
2. **Proper Priorities**: Use dynamic priority calculation based on content importance
3. **Change Frequencies**: Set realistic change frequencies (hourly for breaking news, weekly for articles)
4. **Search Engine Notifications**: Always notify search engines of sitemap updates

### For Performance
1. **Batch Operations**: Process multiple updates together when possible
2. **Async Processing**: Use background jobs for non-critical sitemap updates
3. **Caching**: Implement appropriate caching strategies
4. **Error Recovery**: Handle failures gracefully without breaking the site

## Troubleshooting

### Common Issues

#### Sitemap Not Updating
1. Check database connection
2. Verify webhook endpoints are accessible
3. Check API key configuration
4. Review error logs

#### Search Engines Not Indexing
1. Verify sitemap URLs are accessible
2. Check robots.txt includes sitemap references
3. Ensure proper XML formatting
4. Verify search engine ping responses

#### Performance Issues
1. Check database query performance
2. Review caching configuration
3. Monitor webhook processing times
4. Consider batch processing for bulk operations
